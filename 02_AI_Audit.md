# AI Audit & Governance Framework

## Objetivo

Auditar técnicamente un proyecto desarrollado con asistencia de Inteligencia Artificial.

---

## Principio de Evidencia

Analizá todos los artefactos disponibles del proyecto y limitá tus conclusiones exclusivamente a esa evidencia.

Si algún aspecto no puede verificarse:

> No auditable con la evidencia disponible.

---

## Alcance

La auditoría puede incluir:

- Dashboard
- DAX
- SQL
- Código
- README
- Documentación
- Conversaciones
- Diagramas
- Modelo de datos

---

## Auditoría

Evaluar:

- Calidad técnica
- Consistencia
- Riesgos
- Posibles errores
- Posibles alucinaciones
- Supuestos no explicitados
- Trazabilidad
- Reproducibilidad

Para cada hallazgo indicar:

- Hallazgo
- Evidencia
- Severidad
- Impacto
- Recomendación

---

## Resultado esperado

Emitir un informe ejecutivo con:

- Nivel de confianza
- Riesgo residual
- Recomendación final
