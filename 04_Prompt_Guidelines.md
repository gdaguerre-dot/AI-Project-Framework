# Prompt Engineering Guidelines

Antes de solicitar una tarea indicar siempre:

## 1. Objetivo

¿Qué quiero obtener?

---

## 2. Contexto

¿Qué información necesita la IA?

---

## 3. Alcance

¿Qué debe analizar?

¿Qué debe ignorar?

---

## 4. Restricciones

No inventar información.

No asumir datos faltantes.

Si falta evidencia indicar:

> No auditable con la evidencia disponible.

---

## 5. Formato esperado

Ejemplos:

- Markdown
- Word
- SQL
- DAX
- README
- Tabla

---

## 6. Nivel técnico

Indicar:

- Principiante
- Intermedio
- Profesional
- Consultoría

---

## 7. Validación

Solicitar siempre:

- Posibles errores
- Riesgos
- Mejoras
- Alternativas
