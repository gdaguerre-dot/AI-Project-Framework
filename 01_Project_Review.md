# AI Project Review Framework

## Objetivo

Realizar una revisión profesional del proyecto una vez finalizado.

## Principio de Evidencia

Analizá todos los artefactos disponibles del proyecto y limitá tus conclusiones exclusivamente a esa evidencia.

Si un aspecto no puede verificarse con los documentos suministrados, indicá explícitamente:

> No auditable con la evidencia disponible.

No infieras información no documentada ni presentes supuestos como hechos.

---

## Artefactos que pueden formar parte de la revisión

- README
- Documento Maestro
- Dashboard Power BI
- Modelo de Datos
- Medidas DAX
- Scripts SQL
- Consultas
- Código Python
- Arquitectura
- Diagramas
- Conversaciones relevantes
- Capturas de pantalla
- GitHub

---

## Solicitud de revisión

Actuá como un Project Reviewer Senior.

Analizá exclusivamente los artefactos disponibles del proyecto.

Generá un informe profesional incluyendo:

- Resumen Ejecutivo
- Objetivos alcanzados
- Entregables
- Competencias desarrolladas
- Aprendizajes
- Oportunidades de mejora
- Próximos pasos
- Conclusión final

Todas las conclusiones deberán indicar claramente la evidencia utilizada.
